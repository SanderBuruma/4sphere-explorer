# Roadmap: 4-Sphere Explorer v1

**Project:** Interactive 4D sphere (S³) explorer with 30,000 navigable points, procedural audio, and intuitive controls.

**Core Value:** Make 4D geometry tangible through navigable exploration.

**Phases:** 3
**Granularity:** Coarse
**Coverage:** 10/10 v1 requirements mapped

## Phase Details

### Phase 1: Performance & Navigation Foundation

**Goal:** Enable fast, responsive exploration with practical ways to navigate and revisit locations.

**Depends on:** Nothing (foundation phase)

**Requirements:** PERF-01, NAV-01, NAV-02, NAV-04

**Success Criteria** (what must be TRUE):
1. User can type to search the point list by name and see results instantly filter
2. User can bookmark current position and return to any saved bookmark from a persistent menu
3. User can press Tab to automatically travel to the nearest unvisited visible point in the viewport
4. Visibility queries complete sub-linearly (spatial indexing) without perceptible lag when rotating through dense regions

**Plans:** 3/3 plans complete

Plans:
- [x] 01-spatial-indexing-bookmarks-PLAN.md — KDTree spatial index + bookmark save/restore system (PERF-01, NAV-01)
- [x] 02-search-filtering-PLAN.md — Real-time name search with instant filtering (NAV-02)
- [x] 03-auto-travel-unvisited-PLAN.md — Tab key auto-travel to nearest unvisited point (NAV-04)

---

### Phase 2: Visual Polish & Immersion

**Goal:** Enhance exploration atmosphere and provide visual feedback for movement and location discovery.

**Depends on:** Phase 1

**Requirements:** VIS-01, VIS-02, VIS-03, NAV-03

**Success Criteria** (what must be TRUE):
1. Points near the crosshair render with visible glowing halos that grow brighter as distance decreases
2. Background displays slowly scrolling stars that shift perspective during 4D rotation to convey depth
3. An animated line connects the player position to the current travel target during active movement
4. Visited points appear as a fading trail of dots in the viewport, showing the exploration path taken

**Plans:** 4/4 plans complete

Plans:
- [x] 01-glow-halos-PLAN.md — Distance-based glow halos on points (VIS-01)
- [x] 02-starfield-PLAN.md — Animated parallax starfield background (VIS-02)
- [x] 03-travel-line-PLAN.md — Animated dashed travel line to target (VIS-03)
- [x] 04-breadcrumb-trail-PLAN.md — Fading breadcrumb trail of visited points (NAV-03)

---

### Phase 3: Information & Context

**Goal:** Provide detailed exploration data and session statistics to enhance understanding of navigation and discovery.

**Depends on:** Phase 1 (navigation features enable comprehensive tracking)

**Requirements:** INFO-01, INFO-02

**Success Criteria** (what must be TRUE):
1. User can click any visible point to open a detail panel displaying its 4D coordinates, unique name, distance, and audio timbre/scale parameters
2. User can see a live statistics overlay showing total points visited, cumulative distance traveled in radians, and elapsed session time

**Plans:** 2/2 plans complete

Plans:
- [x] 01-audio-params-PLAN.md — Audio parameter extraction function (INFO-01 dependency)
- [x] 02-radial-menu-detail-panel-PLAN.md — Radial menu + detail panel UI (INFO-01)
